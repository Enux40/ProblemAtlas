# ProblemAtlas — Scoring System

## 1. Purpose

The scoring system helps users quickly evaluate a problem from multiple angles.

It is NOT meant to predict success or guarantee outcomes.

## 2. Scale

All scores range from:

1 → very low  
5 → very high  

## 3. Metrics

### Demand
How often this problem appears in real-world scenarios.

1 → rare  
5 → very common  

---

### Urgency
How painful or costly the problem is.

1 → minor inconvenience  
5 → critical pain point  

---

### Monetization
How realistic it is to charge for a solution.

1 → hard to monetize  
5 → strong monetization potential  

---

### Difficulty
How hard it is to build a working MVP.

1 → very easy  
5 → very complex  

---

### Saturation
How crowded the solution space is.

1 → few existing solutions  
5 → very crowded  

---

### Portfolio Value
How impressive the project is for showcasing skills.

1 → basic  
5 → highly impressive  

---

### Practice Value
How useful the project is for learning.

1 → low learning value  
5 → strong learning value  

---

## 4. Important Notes

- Scores are directional, not precise
- They are based on judgment + evidence
- They should not be over-interpreted

---

## 5. Labels

Use scores to assign readable labels.

Examples:

- Great for beginners
- Strong freelance opportunity
- Good micro-SaaS candidate
- Practice-first project
- Crowded but valuable
- Strong pain, weak monetization

---

## 6. Anti-Patterns

DO NOT:
- combine scores into a fake “total score”
- present scores as scientific truth
- imply guaranteed success

---

## 7. Editorial Rule

Scores must be consistent across entries.

If unsure:
- lower confidence
- explain uncertainty
