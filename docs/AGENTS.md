# ProblemAtlas — Agent Instructions

## 1. Project Overview

ProblemAtlas is a curated software-problem discovery platform.

The product focuses on:
- clarity
- usefulness
- honest evaluation

---

## 2. Core Rules

- Do NOT fabricate demand signals or evidence
- Do NOT introduce features outside PRD scope
- Do NOT overcomplicate solutions
- Do NOT generate fake or low-quality content

---

## 3. Engineering Guidelines

- Use strict TypeScript
- Prefer simple, readable code
- Avoid premature abstraction
- Use server components when appropriate
- Use Zod for validation
- Ensure safe database writes
- Add loading and error states

---

## 4. UX Guidelines

- Keep UI clean and editorial
- Prioritize readability over density
- Avoid visual clutter
- Scores must be easy to understand
- Labels must be human-readable

---

## 5. Data Integrity Rules

- Every problem must have real reasoning behind it
- Evidence must be editable and transparent
- Draft content must never appear publicly

---

## 6. Admin Rules

- Only published problems appear in public queries
- Draft problems remain private
- Editing must not break existing data

---

## 7. SEO & Performance

- Use semantic HTML
- Ensure pages load fast
- Avoid unnecessary client-side rendering
- Maintain SEO-friendly URLs

---

## 8. Code Quality

- Avoid duplication
- Keep components small
- Use clear naming
- Maintain consistency across files

---

## 9. Safety Rules

- Do not expose sensitive data
- Validate all inputs
- Prevent invalid database writes

---

## 10. When Unsure

Prefer:
- simplicity
- clarity
- maintainability

Ask for clarification instead of guessing.

---
